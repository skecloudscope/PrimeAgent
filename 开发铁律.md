# 开发铁律

每一次开发请求都必须先带入以下规则。

1. 不准私自开发 Agno 和前端之间的中间层。所有运行时能力必须使用 Agno 自己的入口和能力。

2. 不准大规模改动 Agno。确实认为需要改 Agno 时，先反复阅读 Agno 原始源码，确认不是已有能力，再把需求记录到 `需求文档.md`。不要改动 Agno 的代码。不要改动 Agno 的代码。不要改动 Agno 的代码。

3. 前端只能适配 Agno 已有能力。能从 Agno AgentOS / Agent / Team / Workflow / Registry / Components / Tools / MCP / Approval / Storage / Trace / Session / Run 透出的，就接上；找不到 Agno 对应能力的 UI，要删除或隐藏。

4. 每次开发结束前必须检查：

- 是否改了 Agno 源码。
- 是否新增了 PrimeAgent 后端或 runtime wrapper。
- 前端是否只调用 Agno 原生 API。
- 是否还能搜索到旧自建层关键字。
- 前端构建是否通过。

5. 不要自己补充假的，不要自己补一层假 runtime， 或者开发mock 内容， 一定不要，  你可以写需求文档里面

6. 测试的时候一定只做测试，不能做修复和开发，你一定要把问题落盘到「需求文档」 这个文件夹下面，并且为其取名字， 一定要研究codex 是怎么处理这类agent 问题的
